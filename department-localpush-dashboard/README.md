# 部门本地推数据多维看板

这是一个可直接部署到 GitHub Pages 的静态网站。

## 文件说明

- `index.html`：入口页面
- `styles.css`：页面样式
- `app.js`：筛选、图表、明细表、目标管理交互
- `data.js`：由 `消耗明细.xlsx` 生成的看板数据
- `data-summary.json`：数据范围与字段摘要

## 部署到 GitHub Pages

1. 新建一个 GitHub 仓库。
2. 上传本目录下的全部文件到仓库根目录。
3. 在仓库 `Settings` -> `Pages` 中选择 `Deploy from a branch`。
4. 分支选择 `main`，目录选择 `/root`，保存。
5. 等待 GitHub Pages 构建完成后访问生成的网址。

## 首页统计口径

- `仪表盘` 和 `数据展示` 的可视化统计仅统计 `投流类别 = 本地推`。
- `AD` 数据不进入可视化统计，但保留在每日明细和月度汇总明细中。
- 所有消耗统计均使用 `非赠款消耗`。
- 可视化金额展示单位为 `万`。
- “昨日”按当前筛选周期内最后一个有本地推数据的日期计算。

## 页面结构

- `仪表盘`：运营分析、预警、喜报、日耗、周耗、业务和端口看板。
- `数据明细`：月度汇总明细和每日数据明细，支持搜索、排序、导出。
- `目标管理`：维护月度消耗目标和新开目标。
- `数据展示`：面向公司内大屏放映，包含时间进度、倒计时、目标达成、全年趋势和排行。

## 目标管理

目标数据保存在当前浏览器的 `localStorage` 中。换浏览器或清除浏览器缓存后，需要重新填写目标。

`消耗目标` 输入单位为 `万`。例如填写 `300`，系统会按 `3,000,000` 参与后台计算，并在页面中显示为 `300万`。

目标管理支持维护每月 `代充值目标` 和 `代运营目标`，输入单位同样为 `万`。保存后会直接进入仪表盘总数据板块和业务目标达成计算。

## 月度汇总商机情况

月度汇总明细中的 `商机情况` 可在页面内手工调整，支持：

- 存量商机
- 代充值新客户
- 代充值新渠道
- 代运营新客户
- 每日新增

手工调整会保存在当前浏览器的 `localStorage` 中，并同步影响首页喜报板块和新开完成统计。

## 数据口径

- 消耗指标使用 `非赠款消耗`。
- `总消耗`、`赠款消耗`在明细中保留。
- 月度汇总由 `日耗` sheet 按季度、月份、项目、商机、投流类别、业务类型、商务、团队聚合补全。
- 新开口径：月度汇总明细中手工选择为非 `存量商机` 的记录计入新开。
- 业务类型修正：`悉美俪人` 在统计与月度汇总展示中统一按 `代运营` 归类。
